Paxiom Feed API

Subscriber-gated Ethereum proof feeds. Each record includes the value, the predicate, the Ethereum proof context, and a Paxiom commitment so downstream systems can verify, archive, and replay the fact.

Live feed access requires an approved subscriber token. Public pages may show schemas, predicates, health, and sample records, but production data is not open-access.

Base URL And Auth

Base URL: https://api.paxiom.org
Header:   x-paxiom-feed-token: <subscriber-token>
curl https://api.paxiom.org/v1/feed/latest?limit=5 \
  -H "x-paxiom-feed-token: $PAXIOM_FEED_TOKEN"

Endpoints

GET /health

Checks whether the API is reachable and whether feed reads require subscriber auth.

GET /v1/predicates

Lists live and planned predicates, proof systems, expected inputs, outputs, and use cases.

GET /v1/feed/latest

Returns recent feed records allowed by the subscriber token. Supports limit, predicate, and subject.

GET /v1/feed/commitment/:id

Looks up a specific proof-backed record by Paxiom commitment or feed item hash.

Predicates

A predicate is a verifiable claim about Ethereum state. The first live package focuses on high-value state facts that trading and risk systems already consume.

storage_equals

Proves a contract storage slot value under an Ethereum state root.

  • Status: live
  • Proof: EIP-1186 Merkle Patricia proof
  • Use: custom protocol accounting and feed construction

uniswap_v3_slot0

Proves and decodes Uniswap V3 pool price/tick state.

  • Status: live
  • Proof: EIP-1186 MPT plus slot0 decoder
  • Use: routing, monitoring, divergence, replayable signals
