Verified Ethereum data feeds

Paxiom turns Ethereum state into subscriber-grade proof records.

The first product is not custody. It is a feed of machine-readable Ethereum facts, verified from Merkle Patricia proofs, committed through Paxiom, and labeled by verification depth.

Live predicateUniswap V3 pool slot0 state.
Current proof levelmpt_verified against Ethereum state roots.
Feed custodyNone. Subscribers consume data, not hosted assets.
Next depthAO commitment, recursive aggregation, then ZK verification.

What Subscribers Receive

Each feed item includes the predicate, subject, value, block number, state root, proof system, proof hash, Paxiom commitment, verification level, timestamp, and optional signature. Subscribers can store the feed item, verify its commitment, and decide how much trust depth their system requires.

Canonical pool state

Verified Uniswap V3 slot0 facts with decoded tick and price fields.

Storage facts

Generic storage_equals records for Ethereum contracts and slots.

Commitment lookup

Feed items can be retrieved by Paxiom commitment or feed item hash.

Verification Levels

Paxiom feed records are explicit about their trust depth. This keeps the product useful now while preserving the path to beaconless recursive verification.

rpc_observedSeen from an RPC source but not independently proven.
mpt_verifiedEthereum account/storage proof verified locally against a block state root.
ao_committedThe verified fact has been submitted to the AO proof-corpus process.
recursive_verifiedThe fact binds to a recursive history corpus.
zk_verifiedThe fact binds to a succinct ZK proof of the relevant history and predicate.

Feed Shape

The feed is built for subscribers who need auditable market state, not custody. A typical item looks like this: 

{
  "schema": "paxiom.feed.v1",
  "feed_id": "ethereum.uniswap_v3_slot0",
  "predicate": "uniswap_v3_slot0",
  "subject": "ethereum:0x88e6...f5640:slot0",
  "block_number": 24961596,
  "state_root": "0xd8f847...",
  "proof_system": "eip1186-mpt+uniswap-v3-slot0",
  "verification_level": "mpt_verified",
  "custody": "none",
  "paxiom_commitment": "775ca05c...",
  "feed_item_hash": "..."
}

Subscriber Endpoints

The local feed server exposes the current subscriber API. It can sit behind an authenticated reverse proxy for early subscribers. 

GET /health
GET /v1/feed/latest
GET /v1/feed/subjects
GET /v1/feed/commitment/:commitment

Paxiom data feeds are not custody, trade execution, or investment advice. They are proof-labeled records that downstream systems can verify and consume according to their own risk policy.
