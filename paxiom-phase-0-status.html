Three gates. One substrate.

Phase 0 closes when the operator runs three commands and each one exits 0. The code is scaffolded in companion repositories, the runbooks are written. What remains is the operator running the documented commands on real infrastructure — and the companion repos becoming publicly browse-able so the evidence is independently auditable.

G-200 · Why this page exists

The discipline of evidence.

The build map (PXM-R-100) flips four items into "What's done" the moment the substrate scaffolding lands. This page is the audit trail underneath those flips: each gate names its acceptance criteria, points at the source of evidence, and quotes the exact command the operator runs to close the live side.

Where a build-map flip says "done", this page says scaffolded plus operator-pending live verification. Both forms of "done" are honest; this is where they're disambiguated.

About the evidence links. The links above point at the companion repos k-luecke/paxiom and k-luecke/bls-verifier. They resolve when those repos are public; if you encounter a 404, the companion is currently private and the public proof is operator-runnable rather than browse-able. The page will tighten its language to "closed" once both the operator-side commands have been run and the companion repos are publicly readable.
