A-120 / S.03 · L1 historical state reconstruction substrate Deferred — substrate decision moved to A-205 spec
No-trust-in-RPC architectural moat for historical state at depth.

Original framing: Load Network state reconstruction tested end-to-end. Updated framing (2026-05-03): Load Network operates as a storage chain (LS3) and its state-reconstruction product (Ledger Archivers via weaveVM/evm-state-reconstructing) covers Metis, RSS3, GOAT, Avalanche C, Dymension, Humanode, Scroll, and phala-mainnet-0 — but not Ethereum L1 mainnet. Paxiom committed to L1; the substrate question is therefore an open product decision, not a Phase 0 implementation gate. Resolution moves to the Service A-205 (Verified Historical State) specification. The MPT proof verification logic in paxiom/load-network/test/verifier.test.mjs remains valid — that logic verifies proofs from any source. Per the no-RPC framing: the load-bearing claim is "no trust in RPC" (local verification against consensus-attested state roots), not "no RPC anywhere as a data source."

What stands
  • paxiom/load-network/ — fixture-driven MPT proof verification (20/20 tests pass; reusable for any source of EL proofs)
  • paxiom/docs/load-network-integration.md
What's open (A-205 substrate decision)
  • Pick interim L1 historical archive: conventional archive RPC (Alchemy/QuickNode/Infura/Chainstack) — acceptable because verification is local — or self-hosted Erigon (~3TB) for self-sovereignty
  • Decide future direction: KYVE-backed adapter (no off-the-shelf path for Ethereum L1 mainnet today; would need a custom EVM adapter), or vendor-locked archive with the trust offset to local verification
  • Repurpose paxiom/load-network/: delete the broken client + keep MPT verifier, OR repurpose as an LS3-backed audit-artifact archival sink ($0.02/GB/mo permanent storage on Arweave via x402 facilitator at hyper-x402.load.network)
Scaffolded — code + tests in companion repo
Operator-pending live verification
Closed — operator-verified end-to-end with evidence file
G-200 · Why this page exists

The discipline of evidence.

The build map (PXM-R-100) flips three substrate items into "What's done" once the operator runs the gate commands and an evidence file is produced. This page is the audit trail underneath those flips: each gate names its acceptance criteria, points at the source of evidence, and quotes the exact command the operator ran (or, for S.03, the open product question).

S.01 and S.02 are closed as of 2026-05-03 — operator-verified end-to-end against real Ethereum mainnet consensus, with timestamped evidence files in paxiom/hyperbeam/bringup/{logs,evidence}/. The evidence files are gitignored by design; they are operator-local proofs whose existence and contents are documented above.

S.03 is deferred, not pending. Load Network's reconstruction products do not cover Ethereum L1 mainnet; the substrate question for L1 historical state moves to the Service A-205 specification rather than blocking Phase 0. The fixture-driven MPT verification logic (20/20 tests pass) survives the deferral — that logic verifies proofs from any source.

About the evidence links. The links above point at k-luecke/paxiom and k-luecke/bls-verifier — both public. PR references (paxiom #2; bls-verifier #3, #5, #6) carry the implementation work that lands on main; this page will be browse-trustworthy once those PRs merge. If any link 404s, the corresponding PR is still open.
